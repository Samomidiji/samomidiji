Rainydays
Saver

A digital savings platform built for clarity and trust.

Overview

Rainydays Saver is a platform that allows customers to make financially responsible actions. They provide different options from which their customers can choose from and these includes joining a Rotational Saving plans where users save in group with other people towards a set goal. Users can also create a Target Saving plans where they save alone and define the mechanism of how they want their saving to go through. Rainydays Saver also provide options to invest and request for loans within a time frame.

Problem

Have done most of their operations manually and via Whatsapp, Rainydays Saver aim to digitize the products (Target Saving, Rotational Saving, Investments and Loans). The had first contacted a product manager with whom they create user flow for the product. I was brought in to develop the User interface and experience of the product. The product was divided into two parts; The admin interface and the customer interface.

Task

I was tasked with creating both the admin and consumer product interface as well as brand consistency for both sections (Admin and Users).

Task for Admin Interface

  • Design a dashboard that would highlight all products and their metrics at a glance
  • Overview of each products with specific informations and decisions
  • Admins should be able to calculate credit score and eligible amount for loanees
  • Preview applications and make decisions to accept or decline
  • Admin should have the ability to flag, suspend, and release accounts

Task for Users Interface

  • Design a dashboard that would highlight which products the user is on.
  • Overview of products with option to select product of choice
  • Provide users the ability to fund their wallet
  • Provide users with the option to make payment either with cards or bank transfer
  • Users should have the ability to schedule or manually make payment

Conclusion

I was able to deliver a crisp user interface for the client while simplifying the user flow they provided. I also carried along the client at regular interval and implement their feedback instantly. This reduced number of iterations and made the whole process faster. Although I would have loved to be fully involve in the development of the User flow and research, I was able to input some my experience to the overall User Experience, which the client acknowledged and appreciated.

Some feedback when testing

  • About 70% of the user found it easy to navigate the product and were clear about each product descriptions
  • 38% of the users complain that the process of applying for both Target and Rotational saving is longer than expected as it takes them from one screen to screen instead of a one page application.
  • The Admin dashboard was a major success as the client was pleased to see their metrics at a glance and their performance in a graphical form
  • 68% of users feel they can trust the brand and save with them as the Interface of the product wasnt tacky.
